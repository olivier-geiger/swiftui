Fondamentaux

#1 — Xcode & SwiftUI

Comment utiliser Xcode dans un projet SwiftUI

Setup

Xcode est l'IDE d'Apple pour développer des apps iOS. Un projet SwiftUI se compose d'une struct qui conforme au protocole View et retourne du contenu via la propriété body. Le canvas en temps réel (Preview) vous permet de voir les changements instantanément.

import SwiftUI struct ContentView: View { var body: some View { Text("Hello, SwiftUI!") } } // Preview dans Xcode #Preview { ContentView() }
#2 — Text

Ajout de texte dans SwiftUI

Composant

Text est la vue de base pour afficher du texte. Il supporte le Markdown natif et de nombreux modifiers pour contrôler la typographie, l'alignement et la mise en forme.

Text("SwiftUI Bootcamp") .font(.largeTitle) // Taille de police system .fontWeight(.bold) // Épaisseur .foregroundColor(.orange) // Couleur du texte .multilineTextAlignment(.center) .lineLimit(2) // Nombre de lignes max .minimumScaleFactor(0.5) // Réduit si nécessaire .italic() // Italique .underline() // Souligné .strikethrough() // Barré .kerning(2) // Espacement lettres

#4 — Colors

Couleurs, UIColor et hexadécimales

Style

SwiftUI propose des couleurs système adaptatives (light/dark mode), des couleurs UIKit via Color(uiColor:) et la possibilité de définir des couleurs via littéraux ou hexadécimaux via une extension.

#5 — Gradients

Style

#7 — Images

Ajouter des images à une application SwiftUI

Composant

Image charge des assets depuis le catalogue, le bundle ou le réseau. .resizable() est indispensable avant tout redimensionnement. AsyncImage gère les images distantes.

State & Gestion des données

#19 — @State

État local d'une vue

Property Wrapper

@State est utilisé pour les données locales d'une vue. Quand la valeur change, SwiftUI re-render automatiquement la vue. Toujours utiliser pour les types valeurs simples.

@State private var count: Int = 0 @State private var isShowing: Bool = false @State private var text: String = "" var body: some View { VStack { Text("Count: \(count)") Button("+") { count += 1 } } }
#22 — @Binding

Lien bidirectionnel

Property Wrapper

@Binding crée une référence vers une valeur @State du parent. Permet à un composant enfant de modifier l'état du parent sans en être propriétaire.

// Vue enfant struct ChildView: View { @Binding var isOn: Bool var body: some View { Toggle("Active", isOn: $isOn) } } // Vue parent @State private var isOn = false ChildView(isOn: $isOn) // Passe le binding avec $

#50 — @ObservableObject / @StateObject

ViewModel partagé

Architecture

Pour iOS 16 et antérieur. Un ObservableObject est une classe dont les propriétés marquées @Published déclenchent une mise à jour des vues abonnées.

class MyViewModel: ObservableObject { @Published var title: String = "Hello" @Published var items: [String] = [] func fetchData() { items = ["Item 1", "Item 2"] } } struct MyView: View { // Crée ET possède le ViewModel @StateObject private var vm = MyViewModel() // Observe sans posséder // @ObservedObject var vm: MyViewModel }
